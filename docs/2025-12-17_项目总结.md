# PestManKill 项目总结 - 2025年12月17日

PestManKill 是一个受“三国杀”启发的网络多人卡牌游戏。它采用了现代 C++23 构建客户端-服务器架构，是一个演示高级 C++ 模式、ECS 架构和异步网络技术的学习项目。

## 总体架构

- **模块划分**：
  - `src/client/`：负责 UI、SDL3/ImGui。
  - `src/server/`：包含权威游戏逻辑和网络功能。
  - `src/shared/`：定义消息协议和公用类型。
- **核心模式**：
  - **ECS**：使用 EnTT 实现实体-组件-系统架构。
  - **网络**：基于 ASIO + C++20 协程 (`awaitable`)。
  - **游戏流**：由 `dispatcher` 事件驱动。

## 快速入口（常查文件）

- **服务端入口**：`src/server/main.cpp`
- **客户端入口**：`src/client/main.cpp`
- **游戏上下文**：`src/server/context/GameContext.h` (包含 `registry`、`dispatcher`、`logger`、线程池)
- **主窗口/界面**：`src/client/view/MainWindow.h`

## 构建 / 运行 (Windows PowerShell 示例)

```powershell
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release; cmake --build build --config Release
# 运行示例
.\build\src\client\Release\PestManKillClient.exe
.\build\src\server\Release\PestManKillServer.exe
```

## 仓库特定约定

- **头文件优先**：大多数实现写在 `.h` 中，除非必要，避免新增 `.cpp` 文件。
- **日志**：使用 `m_context->logger->info/warn/error()` (spdlog)，不要用 `std::cout`。
- **引用风格**：尽量使用项目根相对路径的 `#include "src/..."` 格式。
- **线程安全**：网络确认/重试相关状态 (例如 `m_pendingAcks`) 受 mutex 保护，遵循已有锁策略。

## 关键子系统

- **网络**：`src/server/net/NetWorkManager.h` (可靠发送、接收循环、ACK 机制)、`src/server/net/ClientSessionManager.h` (endpoint ↔ entity 映射、心跳/超时、断线重连)。
- **ECS/系统**：`src/server/systems/`；接口参考 `Interface/ISystem.h` (项目使用 `entt::poly` 做类型擦除/多态)。
- **UI**：`src/client/ui/`、`src/client/view/MainWindow.h` (布局管理器、`Application::run` 渲染循环)。
- **资源嵌入**：`cmake/embed_resource.cmake`，生成头文件位于 `build/embedded/`。
- **协议与常量**：`src/shared/messages/`、`src/shared/common/` (如 `TurnPhase` 枚举)。

## 常见工作流 / 注意事项

- **修改系统行为**：首选编辑 `src/server/systems/` 下对应系统的头文件，使用 `dispatcher.sink<EventType>()` 连接事件处理函数。
- **增加网络消息**：同步更新 `src/shared/messages/` 的消息定义，并在 `NetWorkManager` 中处理序列化/反序列化。
- **添加资源**：将文件加入根 CMakeLists 的 `ASSETS` 列表，构建后引用 `build/embedded/` 产生的头文件。
- **调试会话**：使用 `ClientSessionManager` 的 API (`registerClient`、`updateHeartbeat`、`getPlayerEntity`) 来定位 endpoint 与实体映射问题；心跳超时配置常量名为 `HEARTBEAT_TIMEOUT` 或类似。
